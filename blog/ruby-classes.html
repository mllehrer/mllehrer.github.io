Classes in Ruby

Ruby is an object-oriented programming language; instead of building programs along procedures (if-then-else) alone, it is composed of a set of objects that have data and behavior. So within an object, there may be if-then-elses, and they can interact with other if-then-elses. In this way, programs can model the "real world" as a set of interactive objects, and can function in ways that a procedural program might not. A class is one way to build such objects.

  • A class name should start with a capital letter.
  • Classes can have "instance variables" that remain consistent unless changed throughout the class. These are denoted by an @ in front of the variable name.
  • A class can be set up with these variables using an "initialize" method, like in the example below.
  • Classes can have several methods. As a rule, though, if the methods start to diverge too much from each other, you should continue creating a new class to hold the most differing methods.
  • Methods that can be called throughout the class (the same way instance variables can be) are called "instance" methods.
  • Classes can make a bunch of objects. Let's make a Pokemon! How about Corsola, the cute little guy up top?

    def initialize(name, number, type1, type2=nil) #some pokemon have two types and some only have one
    @name = name
    @number = number
    @type1 = type1
    @type2 = type2
    end
