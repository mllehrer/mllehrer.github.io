Arrays and Hashes

Ruby includes two main ways to make list objects: Arrays and Hashes. Both can store strings, integers, floats, and lists. They can even store other arrays and hashes. More about each below.

Arrays

An arrays is an ordered list of things. Arrays assign an index number to each element inside, starting at 0 and ending in the length of the array-1. You can ask the program to return an element in an array by calling its index. In the array above, my_array[0] would return "These. The index of an array cannot change unless you tell it to switch the order. So let's say we wanted to make our array talk like Shakespeare. Calling my_array[1], my_array[2] = my_array[2], my_array[1] would make the inside of the array read "These objects 4" instead of "These 4 objects."

You can perform an action on one specific element in an array by calling it just on that element's index: my_array[1].to_s would turn the 4 into a string of "4". You can also perform an action on every object in an array by using the or command. my_breakfast = ["eggs", "tortilla", "cheese", "coffee"] my_breakfast.each { |food| puts food } Prints to the console "eggs" "tortilla" "cheese" "coffee" I'm hungry.

Hashes

We can think of hashes as a more customizable version of an array. We haven't actually done that much with them at DBC, so I can't say I know as much about them as I do about arrays. Above I mentioned that every array has an index--array[number of element]--and an element. Hashes instead allow the user to assign keys instead of having a built-in index. Since the keys function as an index, they must be unique within each hash. Having customizable keys allows an extra layer of information within the list. Keys can be symbols like we saw above or they can be strings. Since a symbol is unique within a piece of code, it takes less time to process because ruby doesn't have to keep making sure it means the same thing, like it does with a string.

Let's make a breakfast hash (Oh do I WISH): my_breakfast = { :protein => "eggs" :carbs => "tortilla" :happiness => "cheese" :lifeblood => "coffee" } Great! Now if you want to know what the protein in today's breakfast will be, you can ask it for my_breakfast[:protein] Strings would be a better choice than symbols if we were making a number of breakfast hashes for the days of the week. With a symbol, only one hash can have a :protein key. With a string, you can have "protein" keys for every hash, because you should have protein in every breakfast unless you are a rabbit. And you are not a rabbit. Rabbits can't type.

You'll want to use a hash instead of an array when you need the extra layer of information the keys provide.
